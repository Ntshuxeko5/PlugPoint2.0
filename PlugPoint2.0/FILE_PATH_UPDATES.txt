===============================================================================
                        PLUGPOINT2.0 FILE PATH UPDATES
                           INFINITYFREE HOSTING PREPARATION
===============================================================================

This document outlines all the file path changes made to prepare PlugPoint2.0
for hosting on InfinityFree. All paths have been updated to be relative to
the root htdocs directory.

===============================================================================
                                CHANGES MADE
===============================================================================

1. HEADER FILES UPDATED
-----------------------
Created: includes/header_pages.php
- New header file specifically for pages in the pages/ directory
- Uses relative paths: ../assets/css/style.css, ../assets/js/main.js
- Navigation links use relative paths: pages/products.php, pages/login.php, etc.

Updated: includes/header.php
- Updated for root-level access (when accessed from htdocs root)
- Uses relative paths: assets/css/style.css, assets/js/main.js
- Navigation links use relative paths: pages/products.php, pages/login.php, etc.

2. PAGES FILES UPDATED
----------------------
All files in pages/ directory now use:
- include '../includes/header_pages.php' instead of header.php
- All navigation links are relative to pages/ directory
- Asset paths use ../assets/ prefix

Files updated:
- index.php
- login.php
- register.php
- products.php
- checkout.php
- checkout_callback.php
- And all other pages in the pages/ directory

3. PAYMENT CALLBACK URLS
------------------------
Updated: pages/checkout.php
- Callback URL: http://' . $_SERVER['HTTP_HOST'] . '/pages/checkout_callback.php'

Updated: pages/checkout_callback.php
- Notification link: /pages/seller_dashboard.php

4. ASSET PATHS
--------------
CSS: assets/css/style.css (from root) or ../assets/css/style.css (from pages/)
JS: assets/js/main.js (from root) or ../assets/js/main.js (from pages/)
Images: assets/images/ (from root) or ../assets/images/ (from pages/)

===============================================================================
                                ACCESS PATTERNS
===============================================================================

ROOT LEVEL ACCESS (htdocs/)
- Main entry point: pages/index.php
- Assets: assets/css/style.css, assets/js/main.js
- Uses: includes/header.php

PAGES LEVEL ACCESS (htdocs/pages/)
- All page files: index.php, login.php, products.php, etc.
- Assets: ../assets/css/style.css, ../assets/js/main.js
- Uses: ../includes/header_pages.php

INCLUDES ACCESS
- Database: ../includes/db.php (from pages/)
- Auth: ../includes/auth.php (from pages/)
- Footer: ../includes/footer.php (from pages/)

===============================================================================
                                COMMON ISSUES & SOLUTIONS
===============================================================================

ISSUE: CSS/JS not loading
SOLUTION: Check file permissions (644 for files, 755 for directories)

ISSUE: Images not displaying
SOLUTION: Verify assets/images/ directory is writable (755)

ISSUE: Navigation links broken
SOLUTION: Ensure using correct header file (header.php vs header_pages.php)

ISSUE: Payment callbacks failing
SOLUTION: Verify callback URLs are correct for your domain

ISSUE: Database connection errors
SOLUTION: Update includes/db.php with InfinityFree credentials
